# Каталог персональных memory shard-файлов

Используй структуру примеров в этом каталоге и добавь по одному файлу,
написанному владельцем, для каждой записи manifest с точным именем
`memory-shard-<slug>.md`. `manifest.example.json` показывает нужную форму, но
его демонстрационные ID нужно заменить перед копированием в `manifest.json`.
Runtime-код только читает эти файлы; пользователи Telegram, модель, панель и
`/lobotomy` не могут их изменять.

Каждый файл должен быть кратким и не содержать секретов или фактов, которые
участник не разрешил этому боту запоминать.

Операционная gathered-память не записывается в эти `.md`: для каждого
числового Telegram ID бот поддерживает отдельный долговечный JSON-документ в
Upstash Redis. В него кодом попадают сообщения именно этого отправителя с ID,
`first_name`, текстом, временем и источником. Для изображений туда же
добавляется ограниченный OCR и краткое описание Gemma. Это сделано потому, что
файловая система Vercel после завершения serverless-вызова не является
долговечным хранилищем.
